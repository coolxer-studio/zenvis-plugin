# 调试信息入库服务说明文档

## 服务概述

这是一个用于测试验证场景的数据产生和入库服务，通过配置化的数据推送任务，定期生成模拟的用户事件数据并存储到ClickHouse数据库中。

## 配置文件信息

- **配置文件名**: push_user-event.toml
- **服务名称**: 调试信息入库服务
- **服务描述**: 主要用于测试验证场景的数据产生和入库，数据随机自动生成（每隔5秒）

## 数据流程

该服务采用Vector配置格式，包含三个主要组件：数据源(source)、数据处理(transform)和数据目标(sink)。

### 1. 数据源配置 (Sources)

使用 `demo_logs` 类型生成模拟数据，配置如下：

- **类型**: `demo_logs`
- **格式**: `shuffle` (随机打乱顺序)
- **生成间隔**: 每5秒生成一次
- **预定义事件样本**:

```json
  {"event_type":"login","tags":["登录","认证"]}
  {"event_type":"click","tags":[]}
  {"event_type":"view","tags":[]}
  {"event_type":"delete","tags":["已认证"]}
  {"event_type":"modify","tags":["重要","有风险"]}
 ```
### 2. 数据处理 (Transforms)

使用 `remap` 类型对原始数据进行处理和增强：

```vrl
. = parse_json!(string!(.message))
.id = uuid_v4()
.user = encode_base64(random_bytes(16))
.procid = random_int(100, 110)
.reliability = random_float(0.0, 10.0)
.detail = parse_json("{\"method\":\"POST\",\"path\":\"/v1/orders\",\"query\":\"dry_run=false\"}")
.server_time = format_timestamp!(now(), format: "%Y-%m-%d %H:%M:%S")
```


处理逻辑包括：
- 解析原始JSON消息
- 生成UUID作为事件ID
- 创建Base64编码的随机用户标识
- 生成100-110之间的随机进程ID
- 生成0.0-10.0之间的随机可信度评分
- 添加固定的详细信息
- 添加当前时间戳作为入库时间

### 3. 数据目标 (Sinks)

配置了两个数据目标：

#### ClickHouse数据库存储
- **类型**: `clickhouse`
- **数据库**: `zenvis`
- **表名**: `msg_user_event`
- **端点**: `http://clickhouse-service:8123`
- **认证方式**: basic认证
  - 用户名: `default`
  - 密码: `SFGEfSVVMcUHCBCjKmzJ`
- **其他配置**: 跳过未知字段

#### 控制台输出
- **类型**: `console`
- **编码格式**: JSON
- **用途**: 调试和实时查看生成的数据

## 服务特点

1. **自动化数据生成**: 每5秒自动生成一次模拟用户事件数据
2. **多样化事件类型**: 包含登录、点击、浏览、删除、修改等多种事件类型
3. **真实感数据**: 通过随机函数生成具有真实感的ID、用户标识、进程ID和可信度评分
4. **双重输出**: 同时写入ClickHouse数据库和控制台，便于调试和监控
5. **容错处理**: 配置了跳过未知字段，提高数据写入的稳定性
